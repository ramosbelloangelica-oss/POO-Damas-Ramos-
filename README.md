Juego de Damas en C++ con SFML
Este proyecto académico presenta la construcción de un juego de damas clásico en C++11, apoyado en la librería SFML 2.6.1 para la interfaz gráfica. La idea central es mostrar cómo se puede organizar un programa de manera modular, separando cada clase en archivos .h y .cpp, y aplicando principios de programación orientada a objetos junto con un algoritmo de inteligencia artificial.El juego ofrece dos tipos de jugadores: el humano, que interactúa mediante clics en el tablero, y la CPU, que utiliza el algoritmo Minimax para decidir sus movimientos. Este algoritmo evalúa la diferencia de piezas, simula jugadas posibles y selecciona la más ventajosa. Aunque se implementa con una profundidad de búsqueda de 2, puede ampliarse para aumentar la dificultad y hacer más competitiva la partida.La estructura del proyecto refleja una organización clara: clases como Pieza, Tablero, Jugador y Juego se encuentran en archivos separados, lo que facilita la lectura y el mantenimiento del código. Además, la clase InterfazSFML se encarga de dibujar el tablero, las piezas y los textos en pantalla, ofreciendo una experiencia visual sencilla pero funcional.Para compilar el proyecto se requiere un compilador compatible con C++11, como MinGW 13.1.0, y la librería SFML 2.6.1 correctamente enlazada. El comando de compilación típico es:bashCopiarg++ src/*.cpp -o damas -lsfml-graphics -lsfml-window -lsfml-systemAl ejecutar el programa, el tablero aparece en pantalla y el turno inicial corresponde al jugador rojo. El usuario selecciona una ficha y la mueve a una casilla válida; luego la CPU (jugador azul) responde automáticamente. Las reglas básicas de captura y coronación están implementadas, y el juego finaliza cuando un color se queda sin piezas.
En el desarrollo del juego se emplearon varios contenedores de la STL (Standard Template Library), cada uno con un propósito específico. El std::vector se utilizó para manejar colecciones dinámicas de piezas y movimientos, permitiendo un acceso rápido y flexible. El std::set se aplicó para garantizar la unicidad en ciertos registros, como posiciones válidas o jugadas posibles, evitando duplicados de manera eficiente. Finalmente, el std::map se usó para asociar claves con valores, por ejemplo, relacionando coordenadas del tablero con piezas, lo que facilita búsquedas rápidas y una organización clara de la información. El uso combinado de estos contenedores asegura un código más robusto, legible y eficiente, cumpliendo con los principios de modularidad y buenas prácticas en C++.
En conclusión, este proyecto no solo muestra cómo aplicar POO y modularidad en C++, sino también cómo integrar una librería gráfica y un algoritmo de inteligencia artificial en un mismo entorno. Es un ejemplo práctico de cómo combinar teoría y programación para obtener un resultado funcional y académico.
